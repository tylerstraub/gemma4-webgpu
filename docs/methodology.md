# Methodology: crossLabDiff-first development

The engine's correctness surface is verified by running every kernel
against a HuggingFace Transformers reference (CPU or CUDA, any
precision) and comparing hidden states at documented capture points.
Changes to the engine — whether shader edits, weight-layout changes,
or new optimizations — run the correctness sweep first and the
performance bench second. Divergence is a blocker; tps without
correctness is theater.

## Why numerical equivalence

Generation quality benchmarks (MMLU, HumanEval, perplexity on a held-out
set) are valuable downstream, but they are terrible dev-loop signals
for kernel-level work. They take minutes to run, they measure a thousand
things at once, and a subtle bug — the wrong matmul leading dim, a
skipped normalization, an attention scale off by 16× — can reduce
MMLU by 3 points while still producing fluent-looking output.

Numerical equivalence against a trusted reference detects those bugs in
milliseconds. If the hidden state after layer 7 is cosine=0.41 to HF's,
the kernels in layer 7 are wrong. The bug's location is localized
before you run a single downstream eval.

## The reference chain

```
GGUF (llama.cpp-quantized weights)
   │
   │  CPU dequant to F16 at load
   ▼
WebGPU engine (this repo)
   │
   │  captureHidden(tokenId, position, stopAt) → Float32Array
   ▼
crossLabDiff
   ▲
   │  NumPy .npz with {embed, afterLayer:0..33, logits, ...}
   │
HuggingFace Transformers (Python)
   │
   │  Forward hooks on every module boundary
   ▲
google/gemma-4-E2B-it (same weights via HF's GGUF loader path)
```

The `.npz` files are generated by `reference/pytorch/smoke_test.py`
(uv-managed env) on a canonical prompt at a canonical position. The
two shipped references (`public/ref/smoke.npz` and
`public/ref/layer_0_detail.npz`) are committed to git so out-of-box
verification is a zero-setup operation.

## Capture points

Each capture point is an engine-side halt where the relevant GPU work
buffer is read back as F32. The `CapturePoint` enumeration
(in `src/types.ts`):

| Point | Halt location |
|---|---|
| `embed` | after embedding lookup |
| `plePmProjected` | after `per_layer_model_proj` matmul |
| `pleStage1` | after Stage 1 fuse writes `pleInputs` |
| `preRopeQ`, `preRopeK`, `preRopeV` | after linearQ/K/V, before norm+RoPE |
| `postRopeQ`, `postRopeK` | after `fused_per_head_norm_rope` |
| `attnOut` | after `attn_output`, before `linearAttnOut` |
| `afterLayer:L` | after layer L's residual-stream writes |
| `final` | after final RMSNorm |
| `logits` | after LM head + softcap |

A capture does not advance KV state beyond what the pass naturally
writes to producer-layer caches. Callers running a capture sweep call
`prefillForCapture()` and `resetKVForCapture()` to keep state clean.

## The dev loop

For any change that touches a shader, a matmul dispatch, a weight
upload path, or a normalization:

```js
// 1. Build change, full-reload the browser.

// 2. Load the reference.
await window.lab.loadReferenceTensors('/ref/smoke.npz');

// 3. Run the sweep.
const baseline = {
  embed: 0.997232,
  afterLayer_0: 0.999841,
  afterLayer_33: 0.941056,
  logits: 0.997847,
};
const results = {};
results.embed = (await window.lab.crossLabDiff('embed')).cosineSimilarity;
results.afterLayer_0 = (await window.lab.crossLabDiff({ afterLayer: 0 })).cosineSimilarity;
results.afterLayer_33 = (await window.lab.crossLabDiff({ afterLayer: 33 })).cosineSimilarity;
results.logits = (await window.lab.crossLabDiff('logits')).cosineSimilarity;

// 4. Compare against archival floor.
for (const k of Object.keys(baseline)) {
  console.assert(results[k] >= baseline[k] - 0.001, `regression at ${k}`);
}

// 5. ONLY after correctness is green: bench.
await window.lab.bench('Hello, how are you?', { maxTokens: 64, runs: 5 });
```

If cosine drops by more than ~1e-4, that's a real divergence, not
float noise. Don't paper over it with a tighter tolerance.

## Interpretation of cosine values

- **≥ 0.999** — kernel is bit-equivalent to reference within F16
  rounding. This is the expected value at early layers.
- **0.99 – 0.999** — accumulated F16 rounding error through many
  layers. Expected at deep layers of the stack.
- **0.94 – 0.99** — the "drift floor" inherent to F16 accumulation
  across 33 layers. Pre-finalNorm states live here.
- **0.5 – 0.94** — a single kernel likely has wrong output. Bisect
  across layers to localize.
- **< 0.5** — catastrophic bug (wrong normalization, wrong scale,
  wrong matmul orientation). Almost always reproducible at layer 0.

The archival `afterLayer:34` capture deliberately sits at cos 0.42 —
not a bug, but a documented semantic mismatch (this repo captures
pre-finalNorm, the reference captures post-finalNorm). Don't chase it.

## When crossLabDiff is not enough

Weight-upload verification is a separate concern — you can have bit-
correct kernels compiling against wrong-layout weights and get
garbage hidden states downstream. The `compareWeightSlice`,
`compareWeightRow`, and `probeLayerWeight` verbs dequantize a named
GGUF tensor directly from the on-disk bytes and compare against what
the engine uploaded to GPU. These ran extensively during the
development journey; the reports helped eliminate "did I upload
this right?" as a possibility before hunting shader bugs.

The `probePerLayerModelProjLayout` verb specifically exercises the
F16-packing of `per_layer_model_proj`, which was the largest tensor
transposed during the Gemma 3 → Gemma 4 port.

## What to do when you find a divergence

1. Run the full sweep across every capture point you have. The first
   point where cosine drops below 0.99 is where the bug starts.
2. If the drop is at `afterLayer:L`, bisect: check `attnOut:L`
   (attention block done) and the `preRopeQ/K/V:L` captures.
3. If the drop is pre-RoPE, the bug is in the Q/K/V linear projections
   or the weight upload. Use `compareWeightRow` to verify.
4. If the drop is at `postRopeQ/K`, the bug is in RoPE or the fused
   norm.
5. If the drop is at `attnOut`, the bug is in attention scoring,
   softmax, or attn-output. Don't forget the scale (must be 1.0, not
   1/sqrt(d) — see [`findings/gemma4-attention-quirks.md`](./findings/gemma4-attention-quirks.md)).

## Regenerating the reference

If the HuggingFace implementation changes (rare but real — Transformers
has shipped breaking RoPE and normalization changes mid-minor-version
in the past), the `.npz` files need regeneration:

```sh
cd reference/pytorch
uv sync                                 # install deps (first time)
uv run smoke_test.py                    # regenerate smoke.npz
uv run probe_layer0.py                  # regenerate layer_0_detail.npz
```

The scripts print the positions, cosines, and SHA256 of the output npz
— commit them to `public/ref/` only after validating they reproduce
known-good cosines against the previous engine build.
