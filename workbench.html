Agent-first. The primary interface is the JSON API at window.lab. The buttons above are a minimal human-facing surface; every verb has a programmatic equivalent: 
await window.lab.init('e2b');                   // ~70–90s first load
await window.lab.bench('Hello, how are you?', { maxTokens: 64 });
await window.lab.bench(undefined, { maxTokens: 200 });
await window.lab.loadReferenceTensors('/ref/smoke.npz');
await window.lab.crossLabDiff({ afterLayer: 33 });
const { markdown } = await window.lab.collectDiagnosticReport();
await window.lab.copyToClipboard(markdown);

Once the diagnostic report is copied, open a new GitHub issue with the device-report template and paste. That's the contribution flow — hardware data from real devices lands as structured, parseable issues and rolls into the project's calibration + tuning-profile registry.
